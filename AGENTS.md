# AGENTS.md

이 문서는 `block-blue-check` 익스텐션을 개발/유지보수하는 에이전트(사람/AI)용 운영 가이드입니다.

## 1) 프로젝트 목적

- 트윗 상세 페이지에서 원문 트윗의 `@멘션` 계정을 추출
- 관련 트윗 영역의 작성자/멘션 계정도 추출
- 원문 작성자와 각 트윗 작성자의 자기멘션은 차단 대상에서 제외
- 그중 블루 체크 계정을 UI 자동화로 차단
- 사용자는 버튼 1회 클릭만 수행

## 2) 코드 구조

- `manifest.json`
  - MV3 확장 메타데이터
  - `content_scripts`로 `src/content-script.js`, `src/styles.css` 주입
  - `x.com`, `twitter.com` 호스트 권한 정의
- `src/content-script.js`
  - 페이지 감지, 버튼 주입, 원문/관련 트윗 파싱, 블루 체크 판별, 메뉴 클릭 기반 차단 처리
- `src/styles.css`
  - 버튼/상태 메시지 스타일
- `README.md`
  - 설치/사용/주의사항 문서

## 3) 핵심 동작 흐름

1. 트윗 상세 URL 감지 (`/status/<id>`)
2. 원문 트윗(첫 번째 `article[data-testid="tweet"]`) 멘션 추출 (원문 작성자 제외)
3. 관련 트윗(동일 컬럼 내 후속 `article`) 작성자/멘션 추출 (작성자 자기멘션 제외)
4. 대상 병합 후 중복 제거
5. 화면에서 조작 가능한 작성자만 실행 대상으로 필터링
6. 버튼 렌더링 (`블루체크 멘션 차단`)
7. 블루 체크 작성자의 `더보기` 메뉴에서 `차단` 클릭 + 확인 클릭
8. 결과(성공/실패/이미차단/비파딱스킵) 상태 표시

## 4) 유지보수 포인트 (자주 깨지는 지점)

- DOM 선택자
  - `data-testid="primaryColumn"`, `article[data-testid="tweet"]`, `data-testid="tweetText"` 등이 변경될 수 있음
  - 작성자 파싱용 `data-testid="User-Name"` 변경 가능성 확인 필요
  - 메뉴 버튼(`button[data-testid="caret"]`, aria-label variants) 변경 가능성 확인
  - 메뉴 항목 텍스트(`Block`, `차단`) 및 확인 버튼(`confirmationSheetConfirm`) 변경 가능성 확인
  - 버튼 주입 위치(`div[role="group"]`)가 깨지면 대체 선택자 추가 필요
- 대기/타이밍
  - 메뉴 렌더링 지연 시 `waitForElement` timeout 조정 필요
  - 실행 중 스크롤 위치와 가상화 렌더링 영향 확인

## 5) 변경 작업 원칙

- 기능 추가보다 안정성 우선
  - 실패 시 조용히 무시하지 말고 사용자 상태 메시지 제공
- 무차별 대량 액션 금지
  - 계정별 클릭 간 지연 유지 (`sleep`)
- 선택자/요청 변경 시 README와 함께 업데이트
- UI 자동화 의존성과 정책 리스크를 문서에서 삭제하지 말 것

## 6) 로컬 테스트 체크리스트

- 정상 케이스
  - 원문 멘션/관련 트윗 대상 중 블루 체크가 있을 때 차단 성공 카운트 증가
- 경계 케이스
  - 원문 멘션 없음 + 관련 트윗 대상 없음
  - 원문 작성자가 블루 체크여도 차단 대상에서 제외되는지
  - 관련 트윗에서 작성자 자기멘션이 차단 대상에서 제외되는지
  - 블루 체크 없음
  - 이미 차단된 계정 포함
- 오류 케이스
  - 메뉴/차단 항목 미탐지
  - 확인 다이얼로그 미탐지
- 회귀 점검
  - SPA 라우팅 후 버튼 중복 생성 없는지
  - 버튼/상태 UI가 트윗 화면을 과도하게 가리지 않는지

## 7) 장애 대응 가이드

- 증상: 버튼이 안 보임
  - URL 패턴과 DOM 선택자 변경 여부 확인
- 증상: 전원 실패(모두 실패)
  - 메뉴 버튼/메뉴 항목 선택자 변경 여부 확인
  - 언어 설정(한국어/영어)에 맞는 라벨 탐지 조건 확인
- 증상: 일부만 실패
  - 이미 차단 상태인지, 해당 계정 카드가 실제 화면에 렌더링됐는지 확인
  - 실패 카운트/콘솔 로그 기반 재시도
